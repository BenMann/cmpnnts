grid

The grid comes with three predefined columns: 25%, 33%, 50% and 75%. According to that, class-names are _25, _33, _50 and _75, which need to be wrapped in a grid as seen below. Additionally the paddings between coloumns can be modified, using paddings from the spacing module. When used with paddings, the grid will automatically re-calculate to fit 100% of the width, minus any given padding.

headlines

Headlines reach from h1 through h5 and like all text elements are rendered with system fonts. They come with a predefined margin of 0.7em top and bottom. Headlines don't require any additional classes but can be extended with colors and other components easily.

paragraphs

Paragraphs give you a predefined font-size and line-height, which is more readable than the default on basically every screen. It also let's you align text in every possible way and is best combined with the general component to allow defining the width of paragraphs.

spacing

The spaceing component is very helpful in combination with all other components, as it provides predefined margins and paddings. You cna either choose certain sides of the element that a space should be applied to, or apply it to every side. Margins use the class-name mX and paddings can be set with pX, where X is a number from 0 to 7. The different spacings are as follows:

0 — .25rem — 0.8rem — 1rem — 1.8rem — 3rem — 6rem — 12rem

When used with the grids component, the grid will automatically account for the given padding like this: 


._25.p7 {
  max-width: calc(25% - 24rem);
}

tags

The tags component is a really small set of rules to create inline-block elements with a border. Being inline-block, they will adjust to the screen-size automatically, however you can also combine them with spacing to increase the margin between them. Also putting a hover on the tags seems like a good combination.
